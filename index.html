How does javascript work?

Javascript is a single-threaded script language. So it has single call stack and it can do one thing at a time.

Computers don't understand the source code directly, browsers do. And to do that browsers need an engine. Every browser has its own javascript engine, for example: SpiderMonkey engine for Firefox and Chakra engine for Internet Explorer, but the most popular engine is Google's V8 Engine. The V8 engine is used inside Chrome and Node.js.

These engines responsible for passing the code through -

  • a parser to check if the syntax is correct or not
  • creating an abstract syntax tree
  • converting the source code to machine code
  • running the code

What is the differece between localStorage and sessionStorage?

localStorage and sessionStorage work almost similarly, they store data across browser sessions but they have differece. For example:

Expiration Time: localStorage data has no expiration time, sessionStorage data gets cleared when the page session ends (the page is closed)

Same URL data: Opening multiple tabs/windows with the same URL, creates a new sessionStorage for each tab/window. But LocalStorage remains the same for each tab/window.

Closing a tab: Closing a tab/window ends the session and clears objects in sessionStorage But the objects are saved in localStorage in the browser.

Syntax: localStorage: myStorage = window.localStorage;
sessionStorage: myStorage = window.sessionStorage;

Uses: localStorage is usually used to store in Shopping cart, Bookmark, Web Theme etc.
sessionStorage is usually used to save automatically the contents of a text field, and if the browser is refreshed, no writing/input is lost.
